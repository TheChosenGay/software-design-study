# 编码设计专题

> 写出高质量、可维护、可扩展的代码

## 核心知识点

### 代码质量
- 命名规范与可读性
- 函数单一职责原则
- 圈复杂度控制

### 设计原则（SOLID）
- Single Responsibility - 单一职责
- Open/Closed - 开闭原则
- Liskov Substitution - 里氏替换
- Interface Segregation - 接口隔离
- Dependency Inversion - 依赖倒置

### 设计模式
- 创建型：工厂、单例、建造者
- 结构型：代理、装饰器、适配器
- 行为型：观察者、策略、命令

### 重构技巧
- 提炼函数 / 提炼类
- 以多态取代条件表达式
- 消除重复代码
