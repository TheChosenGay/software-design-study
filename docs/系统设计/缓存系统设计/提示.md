# 缓存系统设计 - 提示

## 思考方向

### 1. 缓存层次
- 本地缓存（L1）vs 分布式缓存（L2）各自适合什么场景？
- 多级缓存的一致性如何保证？

### 2. 三大问题
- **缓存穿透**：查询不存在的 key，如何防止打穿数据库？
- **缓存击穿**：热点 key 过期瞬间，如何防止大量请求打到 DB？
- **缓存雪崩**：大量 key 同时过期，如何错峰？

### 3. 数据一致性
- Cache Aside vs Write Through vs Write Behind 的区别？
- 先更新 DB 还是先删缓存？为什么？

### 4. 大促场景
- 热点数据识别：如何提前预热？
- 本地缓存在大促中的作用？
