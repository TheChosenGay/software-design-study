# 日志系统设计 - 提示

## 思考方向

### 1. 写入链路
- 客户端如何收集日志？SDK、Agent、Sidecar 各有什么优缺点？
- 如何保证高吞吐写入？消息队列的作用是什么？

### 2. 存储选型
- 实时检索用什么存储？Elasticsearch 的倒排索引原理？
- 冷热分层如何实现？S3/OSS 对象存储的成本优势？

### 3. 查询优化
- 如何加速时间范围查询？索引设计要考虑哪些？
- 全文检索和结构化查询的区别？

### 4. 可靠性
- 消息队列如何保证不丢日志？
- 存储层的副本策略？
